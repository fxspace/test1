    Session 2: Code & Demo

    Class syntax

    In python, we can create a class using the following statement class Fruit:. In this case, the class is named Fruit. The next step is to define how do we want to create/instantiate this class. This is done by specifying the constructor. The constructor def __init__(self): is an special function that is called at the creation of the class, and allows us to pre-define or initialize the values of the variables inside a class. Also it allows us to start our class with a known state.

    class Fruit:
    def __init__(self, size: int, name: int):
        self.size = size
        self.name = name
        self.__other = [] # private variable

    def some_function(self):
        print("Hello world")

    If we take a look at the class defined above, we will see a couple of particular things.
    The first thing that has to get our attention is the self word. This word is used to "define" or "tell" python that all the statements that contain self as the first argument, as well as those which are preceded by this statement, belong to the class.

    Another important statement that has to get our attention is the double underscore on the self.__other variable. These two characters (__) define what we call "private" variables. Note that if we use the same character to define a function: def __some_function(self): that function will also be private.

    Inheritance

    We call inheritance to the procedure of wich a class can "get" or "inherit" all the methods and functions from another class. This is specially suitable when we want or we require to extend the functionality of a certain class.

    The procedure/syntax to inherit is defined in the code below:

    class Computer:
    def __init__(self, ram: int = 8, cpu_core: int=8):
        self.ram = ram
        self.cpu_core = cpu_core

class Laptop(Computer):
    def __init__(self, screen: int=15, usb: int=2):
        self.screen = screen
        self.usb = usb

    In the previous code we can observe that Laptop is inheriting all the objects and methods from Computer though the syntax: class Laptop(Computer):.

    As a further comment, in this case we are not initializing/constructing the class Computer within the class Laptop. To do so, we can make use of the following syntax/code.

    class Computer:
    def __init__(self, ram: int = 8, cpu_core: int=8) -> None:
        self.ram = ram
        self.cpu_core = cpu_core

class Laptop(Computer):
    def __init__(self, screen: int=15, usb: int=2):
        super.__init__(self, 12, 4)
        self.screen = screen
        self.usb = usb

    In this case, the super statement refers to the base class (Computer) and it initializes the Computer class.

    Other types of class

    In python we have special types of classes, such as the dataclass.
    This type of class it allows us to create a class and its variables/methods by just defining the name of the variables and its type.

    from dataclasses import dataclass

@dataclass
class person:
    name: str
    surname: str
    age: int
    height: int
if __name__ == "__main__":
    p1 = person("John", "Doe", 30, 180)
    print(p1)

    This type of class is specially useful for creating classes that only contain data.

    Live coding of a program

    Conceptual Specifications:

    Scope of the project:
    We want to create a Tic-Tac-Toe game that can be played by 1 or 2 people

    Features:
    - Has to have the option of 1 or 2 players.
    - If 1 player selected, the AI has to play.

    Constraints:
    - To win, the player has to put its tile on one diagonal. - If a Player cheats, print a message

    Technical Specifications:

    We are going to use 2 classes:
    - The first one is going to represent the board - The second one is going to be the main game class.

    Each time a player makes a move, the rules have to be checked.

    Class functionality/Requirements:
    - The board class constructor should initialize the board. - The board class will have a list that contains the actual board. This variable should be private. - The board class will have two private methods: - __check_winner: checks if the player has won. - __check_rules: checks if the movement is legit. - The board class will have a public method: - locate_tile: allows the player to make the move. - The main game class Game will have: - Main constructor that will set the number of players. - A public function to allow the players to make a move.

    Program Logic:
    - The program iterates on a while True loop until the game ends. - Whenever the player makes a move: - We first check if the player can actually make the move. - Then we check if by making this move, it has actually won the match. - If a player has won, we end the program.

    Implementation
    This code is the in-lecture implementation of the Tic-Tac-Toe game with all the contraints imposed during the lecture.
